## 01 · Polymarket Arbitrage Bot
**Коммерческий заказ · 2026**
`Крипто` `AI` `Telegram-бот`

### Контекст
Конец 2025 года — бум предикшн-маркетов. Появились конкурирующие площадки Polymarket, Kalshi и другие. Клиент хотел автоматически ловить ценовые расхождения на одинаковых событиях между платформами и делать арбитраж.

### Задача
Telegram-бот, который в реальном времени мониторит более 2 000 событий на 4 площадках и находит спреды.

### Что сделал
- Парсинг 4 платформ с заданным интервалом
- AI-матчинг событий по названию (GPT) — без этого ручной поиск совпадений невозможен
- Алерт в Telegram с мгновенным уведомлением при обнаружении спреда

### Результат
> 5 арбитражных возможностей в день со спредом 3–6%. Клиент использовал бот для принятия торговых решений в период пика активности рынка.

## 02 · DeFi Liquidity Automation (Meteora / Solana)
**Коммерческий заказ · 2025**
`DeFi` `Solana` `SaaS` `Python SDK`

### Контекст
На DeFi-протоколе Meteora (Solana) провайдеры ликвидности теряли деньги из-за того, что вручную не успевали переоткрывать позиции при выходе цены из диапазона. Клиент хотел автоматизировать это и продавать как SaaS.

### Задача
Полноценный SaaS-продукт для автоматического управления позициями ликвидности на Meteora.

### Что сделал
- Windows-приложение с лицензированием по ID материнской платы
- Собственный API-сервер: проверка подписок, аналитика, база данных
- Админ-панель для мониторинга всех активных сессий
- Python SDK для взаимодействия с Meteora API (написан с нуля)

### Результат
> Полноценный продукт с нуля — от SDK до клиентского приложения и биллинга. 4 независимых компонента в единой архитектуре.

## 03 · Meteora Liquidity Radar (Chrome Extension)
**Личный проект · open source · 2025**
`DeFi` `Chrome Extension` `Open Source`

### Контекст
DeFi-трейдеры тратили 15–20 минут на ручной поиск подходящих пулов ликвидности на Meteora под конкретный токен. Инструмента для быстрого поиска не существовало.

### Задача
Расширение для Chrome, которое автоматически находит пулы ликвидности под выбранный токен.

### Что сделал
- Интеграция с GmGn — тянет тикер и адрес токена одним кликом
- Автоматический поиск всех доступных пулов на Meteora
- Фильтрация по параметрам фарминга

### Результат
> Поиск нужного пула — секунды вместо 20 минут. Расширение опубликовал крупный DeFi-блогер у себя в канале.

## 04 · AI Trend Watcher + Автопостинг для SMM-агентства
**Коммерческий заказ · 2026**
`AI` `n8n` `SMM` `Автоматизация`

### Контекст
SMM-агентство теряло клиентов из-за конкуренции с ChatGPT. Их текущий автопостинг на 20 аккаунтов стоил 6 500 руб/мес и регулярно давал сбои. Нужна была AI-трансформация процессов.

### Задача
Заменить сломанный автопостинг и добавить AI-мониторинг трендов для опережения конкурентов.

### Что сделал
- Миграция 20 аккаунтов на Late.dev API через n8n
- Notion Database как центр управления контентом (Notion → n8n → Late.dev → Telegram/X/LinkedIn)
- Telegram-алерты при любых ошибках постинга
- AI Trend Tracker: парсинг 10 источников раз в день, GPT-кластеризация тем, алерт если тема в >50% источников

### Результат
> Трендвотчер используется ежедневно — агентство публикует контент на опережение и набирает больше охватов. Автопостинг работает без сбоев.

## 05 · KofreKo — Система лояльности для HoReCa
**Собственный продукт · в работе · 2025**
`B2B SaaS` `HoReCa` `AI` `Telegram-бот`

### Контекст
Кофейни и рестораны теряют постоянных гостей из-за отсутствия нормальной аналитики и программ лояльности. Существующие решения дорогие и сложные для небольших заведений.

### Задача
MVP системы лояльности с аналитикой, интегрированной в существующие POS-системы.

### Что сделал
- Полноценный продукт за 2 месяца — от идеи до первого клиента
- Интеграция с iiko (популярная POS-система для HoReCa)
- Веб-панель администратора с аналитикой
- Telegram-бот для гостей заведения
- Разработка полностью на AI-инструментах — отсюда скорость

### Результат
> Первый платящий клиент подключён. Продукт в активной итерации. Замеряем метрики до/после внедрения.
